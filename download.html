Installation from Python sources

Linux
Install dependencies: sudo apt-get install python3-pyqt5 libsecp256k1-dev python3-cryptography
Download package: wget https://download.electrum.org/4.5.5/Electrum-4.5.5.tar.gz
Verify signatures: wget https://download.electrum.org/4.5.5/Electrum-4.5.5.tar.gz.asc
gpg --verify Electrum-4.5.5.tar.gz.asc
Run without installing: tar -xvf Electrum-4.5.5.tar.gz
python3 Electrum-4.5.5/run_electrum
Install with PIP: sudo apt-get install python3-setuptools python3-pip
python3 -m pip install --user Electrum-4.5.5.tar.gz

How to verify GPG signatures

GPG signatures ensure that files are signed by the rightful owner of the signing key. If the website were compromised and malicious files were uploaded, the signature verification would fail, as the attacker wouldn’t be able to generate valid signatures. It's important to note that attackers can create valid hashes, which is why we don’t publish binary hashes here, as they don’t provide additional security.

To verify GPG signatures, you need to import the public key of the signer. Electrum binaries are signed with ThomasV's public key. On Linux, you can import the key with this command: gpg --import ThomasV.asc. After importing, always verify the key's fingerprint using independent sources, such as here or through the Web of Trust.

Notes for Windows users

Electrum binaries are often flagged by various antivirus programs, but this is a common false positive due to the heuristics they use. Please avoid reporting this issue to us. If you trust the developers, you can safely verify the GPG signature and ignore the antivirus warnings. If you have doubts, you can either build the binaries yourself or run the software from source. For those concerned about malware, using an operating system that doesn’t rely on antivirus software may be a better option.

Older versions of Windows might require the KB2999226 update to run Electrum.
